## 当前流程架构

```
用户请求
   ↓
1. 题目智能分析 (FrameworkGenerator)
   ├─ 识别题目类型
   ├─ 生成综述框架
   └─ 生成搜索查询 (最多8个)
   ↓
2. 文献搜索 (ScholarFlux)
   ├─ 中文 → AMiner
   ├─ 英文 → OpenAlex + Semantic Scholar
   ├─ 支持组合搜索模式 (title + keyword)
   └─ 并发搜索多个数据源
   ↓
3. 补充搜索 (如需要)
   ├─ 总数 < 20 → 扩大年份范围补充
   └─ 仍为空 → 宽泛搜索 (20年)
   ↓
4. 去重 (seen_ids)
   └─ 按 paper.id 去重
   ↓
5. 提取主题关键词
   └─ 从框架中提取 relevance_keywords
   ↓
6. 筛选文献 (构建候选池)
   ├─ 初始筛选: target_count * 2
   ├─ 按年份、语言、关键词筛选
   └─ 宽松机制: 不满足则放宽条件
   ↓
7-9. 生成综述并验证 (最多2次)
   ├─ 7. 生成综述 (DeepSeek)
   ├─ 8. 验证质量
   │   ├─ 引用数量
   │   ├─ 近5年占比
   │   └─ 英文占比
   └─ 不达标 → 扩大候选池重试
   ↓
10. 检查并修复引用序号
    ├─ 检测顺序错误
    ├─ 自动修复
    └─ 同步更新参考文献
   ↓
11. 计算统计信息并保存
```
