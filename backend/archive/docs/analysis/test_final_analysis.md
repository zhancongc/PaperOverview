## 问题分析

### 1. 已修复的问题 ✅

**问题：** 中文关键词没有被翻译成英文
- 原因：英文翻译查询在列表后面，每轮只使用5个查询
- 修复：将英文查询移到前面，优先使用

**验证：**
```

  - 英文查询: 9 个（优先）
  - 其他查询: 9 个
```

### 2. 当前存在的问题 ⚠️

**问题：** OpenAlex API 返回的文献质量不够高
- 相关文献比例只有 14.4%
- 大量不相关文献（生物信息学、化学、医学等）

**原因分析：**
1. OpenAlex 的搜索算法基于关键词匹配，不够智能
2. 搜索查询虽然正确，但 OpenAlex 返回了包含部分关键词但不相关的文献
3. 例如：搜索 "array-based software verification" 可能返回了 "array-based" 在生物信息学中的应用

**数据库中的相关文献：**
```
数据库中的符号执行相关文献: 17 篇
- Type and interval aware array constraint solving for symbolic execution
- Accelerating array constraints in symbolic execution
- SymQEMU: Compilation-based symbolic execution for binaries
等等
```

## 解决方案建议

### 短期方案

1. **增强相关性过滤**
   - 在过滤阶段，使用 LLM 更严格地检查文献相关性
   - 降低相关性阈值，只保留高度相关的文献

2. **使用多个数据源交叉验证**
   - 只保留在多个数据源中都出现的文献
   - 提高文献的可靠性

3. **添加领域限定**
   - 在搜索时明确限定领域（Computer Science - Software Engineering）
   - 过滤掉明显不相关的领域（Biology, Chemistry, Medicine）

### 长期方案

1. **改进搜索策略**
   - 使用更精确的搜索查询（如 "symbolic execution AND software verification"）
   - 添加排除词（如 -biology -medicine -chemistry）

2. **使用更好的数据源**
   - 考虑使用 Google Scholar、IEEE Xplore、ACM Digital Library
   - 这些数据源在计算机科学领域的文献质量更高

3. **添加文献质量评分**
   - 综合考虑引用数、发表 venue、作者等因素
   - 优先选择高质量文献
