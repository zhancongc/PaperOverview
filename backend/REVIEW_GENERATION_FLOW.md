# 文献综述生成流程

## 架构概览

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          文献综述生成系统                                     │
│                                                                             │
│  用户输入题目 → 智能分析 → 文献搜索 → 文献筛选 → 综述生成 → 验证反馈 → 导出  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

---

## 1. API 入口层 (`main.py`)

### 1.1 核心接口

| 接口 | 方法 | 功能 |
|------|------|------|
| `/api/smart-generate` | POST | 智能生成文献综述（主流程） |
| `/api/classify-topic` | POST | 题目分类 |
| `/api/smart-analyze` | POST | 智能分析 |
| `/api/validate-review` | POST | 验证参考文献质量 |
| `/api/records` | GET | 获取生成记录 |
| `/api/records/export` | POST | 导出 Word 文档 |

### 1.2 请求参数

```python
class GenerateRequest:
    topic: str              # 论文主题
    target_count: int = 50  # 目标文献数量
    recent_years_ratio: float = 0.5  # 近5年占比
    english_ratio: float = 0.3       # 英文文献占比
```

## 2. 智能分析层 (`services/hybrid_classifier.py`)

### 2.1 题目分类流程

```
题目输入
    │
    ▼
┌─────────────────────────────────────┐
│   模式识别分类（优先级最高）          │
│   - 基于XXX的实证研究 → 实证型        │
│   - X对Y的影响 → 实证型               │
│   - 评价与提升 → 评价型               │
└─────────────────────────────────────┘
    │ 未匹配
    ▼
┌─────────────────────────────────────┐
│   关键词分类                         │
│   - 评价型关键词 → 评价型             │
│   - 应用型关键词 → 应用型             │
│   - 实证型关键词 → 实证型             │
│   - 理论型关键词 → 理论型             │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│   LLM 验证（可选）                   │
│   - 验证提取的关键词                 │
│   - 优化关键词表述                   │
└─────────────────────────────────────┘
```

### 2.2 题目类型

| 类型 | 综述结构 | 检索策略 |
|------|----------|----------|
| **应用型** | 三圈交集式 | 研究对象 + 优化目标 + 方法论 |
| **评价型** | 金字塔式 | 评价理论 + 维度指标 + 方法技术 |
| **实证型** | 问题-方案式 | 变量测量 + 影响机制（中英文混合） |
| **理论型** | 溯源式 | 理论起源 + 发展脉络 |
| **通用型** | 通用结构 | 研究现状 + 发展趋势 |

### 2.3 搜索查询生成

**实证型特殊处理**（针对 OpenAlex 优化）：
1. **拆分搜索**：将复合关键词拆分，分别生成英文查询
2. **关键词翻译**：中文关键词自动翻译为英文术语
3. **交叉筛选**：为每个自变量-因变量组合生成影响关系查询

## 3. 文献搜索层 (`services/scholarflux_wrapper.py`)

### 3.1 数据源架构

```
ScholarFlux (统一搜索接口)
    │
    ├── 检测查询语言
    │
    ├─ 中文查询 ───→ AMiner (1 req/s)
    │
    └─ 英文查询 ───→ OpenAlex (5 req/s)
                    └── Semantic Scholar (0.1 req/s)
```

### 3.2 语言检测

```python
def _contains_chinese(text: str) -> bool:
    return bool(text and any('\u4e00' <= char <= '\u9fff' for char in text))
```

### 3.3 智能路由

| 查询类型 | 数据源 | 原因 |
|----------|--------|------|
| 中文关键词 | AMiner | 中文文献覆盖好 |
| 英文关键词 | OpenAlex | 英文文献覆盖广 |
| 补充搜索 | Semantic Scholar | 兜底数据源 |

## 4. 文献筛选层 (`services/paper_filter.py`)

### 4.1 筛选流程

```
原始文献池
    │
    ▼
┌─────────────────────────────────────┐
│   相关性评分                         │
│   - 标题匹配关键词                   │
│   - 摘要匹配关键词                   │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│   综合评分排序                       │
│   - 相关性得分 50%                   │
│   - 被引量得分 30%                   │
│   - 新近度得分 20%                   │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│   比例控制筛选                       │
│   - 近5年文献占比                    │
│   - 英文文献占比                     │
└─────────────────────────────────────┘
    │
    ▼
候选文献池 (target_count × 2)
```

### 4.2 评分算法

```python
# 相关性评分
title_score = 关键词匹配数 × 10
abstract_score = 关键词匹配数 × 5
relevance_score = (title_score + abstract_score) / (标题长度 + 摘要长度)

# 综合评分
final_score = (
    relevance_score * 0.5 +
    citation_score * 0.3 +
    recency_score * 0.2
)
```

## 5. 综述生成层 (`services/review_generator.py`)

### 5.1 生成流程

```
候选文献池
    │
    ▼
┌─────────────────────────────────────┐
│   构建 Prompt                       │
│   - 系统提示词（写作要求）           │
│   - 用户提示词（主题+文献列表）       │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│   DeepSeek API 生成                 │
│   - model: deepseek-chat            │
│   - temperature: 0.7                │
│   - max_tokens: 6000                │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│   引用数量验证                      │
│   - 检查引用文献数量                 │
│   - 不足则补充引用（最多重试2次）     │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│   引用后处理                        │
│   - 提取被引用文献                   │
│   - 按出现顺序重新编号               │
│   - 限制每篇文献引用次数≤2           │
└─────────────────────────────────────┘
    │
    ▼
综述内容 + 参考文献
```

### 5.2 写作要求

- **引用格式**：[序号]
- **引用顺序**：从[1]开始连续编号
- **引用数量**：至少 50 篇
- **引用分布**：每篇文献引用次数 ≤ 2
- **字数要求**：3000-5000 字
- **结构要求**：引言 + 主体 + 结论

## 6. 验证反馈层 (`services/reference_validator.py`)

### 6.1 验证指标

| 指标 | 要求 | 验证逻辑 |
|------|------|----------|
| 引用数量 | ≥ target_count | 统计正文中唯一引用编号数量 |
| 近5年占比 | ≥ recent_years_ratio | 计算近5年文献数量占比 |
| 英文占比 | ≥ english_ratio | 计算英文文献数量占比 |
| 引用顺序 | 连续编号 | 检查编号是否从1开始连续 |

### 6.2 验证-重试循环

```
┌─────────────────────────────────────────────────────────┐
│                     生成-验证循环                        │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  第1次生成                                                │
│  ├─ 生成综述                                              │
│  ├─ 验证指标                                              │
│  ├─ 全部通过 → 成功                                       │
│  └─ 未通过 → 扩大候选池重试                               │
│                                                          │
│  第2次生成（如果需要）                                    │
│  ├─ 扩大文献搜索范围（15年）                              │
│  ├─ 重新筛选候选池（target_count × 3）                   │
│  ├─ 重新生成综述                                          │
│  └─ 验证指标                                              │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

## 7. 数据持久化层

### 7.1 记录保存

```python
class ReviewRecord:
    id: int
    topic: str                    # 论文主题
    target_count: int             # 目标文献数量
    recent_years_ratio: float     # 近5年占比
    english_ratio: float          # 英文占比
    review: text                  # 综述内容
    papers: json                  # 被引用文献列表
    statistics: json              # 统计信息
    status: str                   # success/failed
    error_message: text           # 错误信息
    created_at: datetime
```

### 7.2 统计信息

```python
{
    "total": 50,              # 总文献数
    "recent_years_count": 25, # 近5年文献数
    "recent_years_ratio": 0.5,# 近5年占比
    "english_count": 15,      # 英文文献数
    "english_ratio": 0.3,     # 英文占比
    "avg_citations": 28.5,    # 平均被引量
    "year_distribution": {...},# 年份分布
    "journal_distribution": {...}# 期刊分布
}
```

## 8. 导出层 (`services/docx_generator.py`)

### 8.1 导出格式

- **格式**：.docx (Word 文档)
- **文件名**：`文献综述-{主题}-{时间戳}.docx`
- **内容结构**：
  1. 综述标题
  2. 统计信息摘要
  3. 综述正文
  4. 参考文献（GB/T 7714-2015 格式）

## 10. 配置要求

### 10.1 环境变量

```bash
# DeepSeek API（必需）
DEEPSEEK_API_KEY=sk-xxx
DEEPSEEK_BASE_URL=https://api.deepseek.com

# AMiner API（可选，用于中文文献）
AMINER_API_TOKEN=eyJxxx
AMINER_TOKEN_EXPIRES_AT=2026-07-10

# 数据库配置
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=security
DB_NAME=paper
```

### 10.2 Token 过期管理

AMiner Token 有效期约 100 天，系统会在初始化时自动检查：
- 剩余 > 10 天：正常显示
- 剩余 ≤ 10 天：发出警告
- 已过期：提示重新获取

获取地址：https://www.aminer.cn/open/board?tab=control

## 12. 已知优化点

1. **实证型搜索优化**：
   - 中文关键词自动翻译为英文
   - 复合关键词拆分搜索
   - 针对领域生成特定查询（如 media coverage measurement）

2. **引用质量优化**：
   - 自动补充引用（如果数量不足）
   - 按出现顺序重新编号
   - 限制每篇文献引用次数

3. **验证反馈循环**：
   - 验证不通过时自动扩大候选池
   - 最多重试 1 次
   - 最终返回完整验证结果
