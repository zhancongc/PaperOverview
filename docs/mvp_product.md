文献综述生成器 MVP 技术方案
基于你的需求——快速实现后投放到小红书引流，我为你设计一个轻量、低成本、可演示的 MVP 技术方案。核心思路是：用现成的 API 服务快速搭建，避免从零造轮子，3天内跑通核心流程。

一、MVP 核心定位
一句话描述：输入论文主题，自动生成带 50 篇参考文献（近 5 年 50% + 英文 30%）的文献综述。

MVP 目标：

验证用户需求是否真实存在

在小红书上获取第一批种子用户

收集用户反馈，为后续迭代做准备

技术选型原则：

优先使用现成 API，减少开发量

后端简单，能跑通就行

前端够用，能展示结果即可

二、整体架构
text
用户输入（论文主题）
       ↓
┌──────────────────────────────────────┐
│  1. 文献检索模块（Semantic Scholar API）│
│     - 检索相关文献                      │
│     - 按年份/语言/被引量筛选            │
└──────────────────────────────────────┘
       ↓
┌──────────────────────────────────────┐
│  2. 文献筛选模块（本地计算）             │
│     - 确保50篇                         │
│     - 近5年占比 ≥ 50%                  │
│     - 英文文献占比 ≥ 30%               │
│     - 按被引量排序取Top                │
└──────────────────────────────────────┘
       ↓
┌──────────────────────────────────────┐
│  3. 综述生成模块（LLM API）             │
│     - 按学术综述框架生成内容            │
│     - 嵌入文献引用                      │
│     - 输出 Markdown 格式               │
└──────────────────────────────────────┘
       ↓
      输出：综述正文 + 50篇参考文献（含DOI）
